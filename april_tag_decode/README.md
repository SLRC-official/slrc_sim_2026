# Low-memory AprilTag (`tagStandard52h13`)

Use **tagStandard52h13** AprilTags on a **Raspberry Pi** or **Ubuntu** PC without running out of RAM. The trick is to use only **1 bit** of error correction when decoding (a setting called **`maxhamming=1`** in Python). That matches what the low-memory demos in this repo do.

---

## What’s in this folder

| Path | What it is |
|------|------------|
| `apriltag/` | AprilTag 3 source code ([AprilRobotics/apriltag](https://github.com/AprilRobotics/apriltag)). You build it once; it produces a shared library and a Python module. |
| `apriltag/build/` | **Generated by you** — listed in [`.gitignore`](.gitignore) so build files are not committed. Contains `libapriltag.so` and a file like `apriltag.cpython-311-….so` (the exact name depends on your Python version and computer). |
| `april_tag/source_apriltag_env.sh` | Small script you **source** so Python can find those files. |
| `april_tag/sample_apriltag_detect.py` | Short example: load a picture (or a blank image) and print detections. |
| `april_tag/test_52h13_lowmem.py` | Full demo: webcam or video file + on-screen overlay (same idea as the C version). |
| `april_tag/test_52h13_lowmem.c` | Same demo in C++ (optional). |

## Part 1 — Python (use this first)

### 1. Install packages (Ubuntu / Raspberry Pi OS)

Open a terminal and run:

```bash
sudo apt update
sudo apt install -y git cmake build-essential libopencv-dev python3-dev python3-numpy python3-opencv
```

`python3-opencv` is optional if you use `pip install opencv-python` inside a virtual environment instead.

### 2. Build AprilTag

From the folder that contains `apriltag` (this project’s root):

```bash
cd apriltag
mkdir -p build
cd build
cmake .. -DCMAKE_BUILD_TYPE=Release
make -j2
cd ../..
```

If CMake prints a warning that Python bindings were skipped, install `python3-dev` and `python3-numpy`, then delete the `build` folder and run `cmake` / `make` again.

After a successful build, you should have:

- `apriltag/build/libapriltag.so`
- `apriltag/build/apriltag.cpython-….so` (the exact name includes your Python version, e.g. `apriltag.cpython-311-arm-linux-gnueabihf.so` on a 32‑bit Pi with Python 3.11. This name will differe for different hardware architectures. ... is a placeholder)

You only need to **remember the folder** `apriltag/build/`, not the long filename.

### 3. Tell Python where to look (do this in every new terminal)

Environment variables do **not** carry over when you open a **new** terminal tab or window. Each time you start fresh, run the setup again **or** put the same lines in your robot’s launch script (see below).

**Easy way — use the script in this repo** (adjust the path to match where you cloned this project):

```bash
cd ./april_tag
source ./source_apriltag_env.sh
```

If your `apriltag` folder is not next to `april_tag/`, open `source_apriltag_env.sh` in a text editor and set `APRILTAG_ROOT` to the full path of your `apriltag` directory, then save and run `source` again.

**What this does:** it adds `apriltag/build` to `PYTHONPATH` (so `import apriltag` works) and to `LD_LIBRARY_PATH` (so the system can load `libapriltag.so`).

**Check:**

```bash
python3 -c "import apriltag; print('OK')"
```

### 4. Run the sample scripts

Still in a terminal where you already ran `source ./source_apriltag_env.sh`:

**Small test** (optional image file):

```bash
cd /path/to/april_tag_decode
python3 april_tag/sample_apriltag_detect.py
python3 april_tag/sample_apriltag_detect.py /path/to/picture.png
```

**Full demo** (video file or camera index `0`):

```bash
cd /path/to/april_tag_decode/april_tag
python3 test_52h13_lowmem.py april_test.mp4
python3 test_52h13_lowmem.py 0
```

Press **q** or **Esc** to quit the window.

### 5. Use this inside your robot codebase

**Pattern:** build the detector **once** when your node starts, then call **`detect()`** on every new camera frame (grayscale, `uint8`).

```python
from apriltag import apriltag
import cv2

detector = apriltag(
    "tagStandard52h13",
    threads=3,
    maxhamming=1,
    decimate=10.0,
    blur=0.0,
    refine_edges=True,
)

# Each frame:
gray = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)  # if frame is color
results = detector.detect(gray)
for d in results:
    tag_id = d["id"]
    # d["center"], d["lb-rb-rt-lt"], etc.
```

**Environment:** before starting your robot code (manually or from a launch file), run the same **`source …/source_apriltag_env.sh`** as in step 3, or copy the two `export` lines from that script into your startup script. If you use **VS Code**, **Cursor**, or **ROS 2 launch files**, add the same environment variables there so you do not depend on an interactive terminal.

**Optional:** set `APRILTAG_BUILD` to the full path of `apriltag/build` and use the logic in `sample_apriltag_detect.py` to add that path to `sys.path` at startup. You still need `LD_LIBRARY_PATH` (or a system install) so `libapriltag.so` loads.

**Python version:** build AprilTag with the **same** `python3` you use to run the robot (including inside a venv). If you change Python versions, rebuild the `apriltag/build` folder.

## Part 2 — C++ demo (optional)

Only if you need the original C++ test program.

Install build tools (if not already):

```bash
sudo apt install -y g++ libopencv-dev
```

Build the AprilTag library (same as Part 1, step 2). Then compile:

```bash
cd april_tag
g++ test_52h13_lowmem.c -o test_52h13_lowmem \
  -I../apriltag \
  -I../apriltag/common \
  -L../apriltag/build \
  -lapriltag \
  $(pkg-config --cflags --libs opencv4)
export LD_LIBRARY_PATH="$(pwd)/../apriltag/build:${LD_LIBRARY_PATH}"
./test_52h13_lowmem test.mp4
```

The file is named `.c` but must be compiled with **`g++`** because it uses OpenCV.

## Tips

- **Lighting:** low correction (`maxhamming=1`) needs reasonably clear prints and lighting.
- **Speed:** lower camera resolution or skip frames if the Pi is slow.
- **No camera:** test with a recorded video that shows tags.
